# 코드 검토 프로토콜

## 핵심 규칙

"확인 부탁해" / "검토해줘" / "리뷰해줘" = **읽기 전용**. 예외 없음.

- 파일 읽기(Read, Grep, Glob)만 허용
- 코드 수정(Edit, Write) 금지
- 빌드 / 테스트 / 실행(Bash) 금지
- 발견한 문제 → 보고만. 수정은 Director 승인 후 별도 작업

## 검토 결과 형식

표 형태로 ✅ / ⚠️ / ❌ 명확히 구분하여 보고.

```
| 항목 | 상태 | 설명 |
|---|---|---|
| 패턴명 | ✅ 완료 | ... |
| 이슈명 | ⚠️ 주의 | ... |
```

## 배경

Director는 여러 에이전트에서 작업 분리 워크플로우 사용.
검토 에이전트가 코드를 건드리면 검토 목적 훼손.
